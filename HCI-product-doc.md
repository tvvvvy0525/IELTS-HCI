# IeltsCoach —— 免费雅思全科备考平台
## 人机交互课程结课项目 · 产品需求文档

**版本**：v1.0

**适用课程**：人机交互（Human-Computer Interaction）

## 一、产品概述

### 1.1 背景与动机

雅思考试（IELTS）是全球最主流的英语水平认证之一，中国每年报考人数超过 180 万。然而，备考市场高度商业化：

- 正价网课（新东方、雅思哥、British Council）：¥2,000 ～ ¥15,000/套
- 全科 AI 刷题平台（e.g., 墨墨、百词斩 Pro）：¥399 ～ ¥999/年
- 一对一口语陪练：¥150 ～ ¥500/小时

对于经济条件有限的在校大学生而言，系统备考的财务门槛本身就是一道难关。而免费的备考资源则高度碎片化，分散在 YouTube、Reddit、微信群、各类论坛，缺乏统一的学习闭环。

**IeltsCoach** 的核心立场：**以开源 + 免费的姿态，为普通学生提供接近商业级品质的雅思全科备考体验。** 产品覆盖听、说、读、写四科，以 AI 驱动反馈取代人工教学中的高价环节，用间隔复习算法取代昂贵的记忆类课程。

> 实现状态注记（2026-05-09）：当前已稳定支持 `Reading`、`Listening` 和 `Writing MVP`；其中 `Writing` 已具备练笔与反馈基础能力，但正式题库、统计闭环和 Speaking / Vocabulary 仍在后续阶段。

### 1.2 产品定位

| 维度 | 内容 |
|------|------|
| 产品类型 | 桌面 Web 应用（本地优先，无需注册） |
| 核心用户 | 18～26 岁在校大学生，目标雅思 5.5 ～ 7.5 |
| 核心价值主张 | **免费 · 全科 · 闭环** |
| 商业模式 | 完全免费（开源），AI 能力通过本地模型或免费网页 API 实现，零额外支出 |
| 竞争策略 | 不卖课，不订阅；以交互设计和用户体验建立差异化 |

### 1.3 四科功能全貌

```

├── 🎧 听力专项    模拟真题 · 精听跟读 · 分 Part 统计
├── 📖 阅读专项    模拟真题 · 生词即时捕获 · 定位练习
├── ✍️ 写作专项    限时练笔 · 反馈 MVP · 题库/范文待补齐
├── 🎙️ 口语专项    AI 口语教练 · 发音评分 · Part 1/2/3 话题库  ← 新增

```

### 3.2 核心痛点深度分析

---

#### P2 · 收费壁垒：备考资源的阶级门槛

**现象**：目标 6.5 的学生，若要系统备考，面临的典型支出：

| 资源类型 | 典型产品 | 价格 |
|---------|---------|------|
| 全科视频课 | 新东方雅思强化班 | ¥3,980 |
| AI 刷题平台 | 雅思哥 VIP | ¥499/年 |
| 口语陪练（月） | 外教 1 对 1 × 8 次 | ¥1,200+ |
| 写作批改（月） | 达人改作文 × 8 篇 | ¥800+ |
| **合计** | | **¥6,000 ～ ¥10,000** |

**根因**：
- 口语和写作的个性化反馈传统上必须依赖人工，成本刚性高
- 免费内容（YouTube、B 站）缺乏系统性和互动性
- 大模型和语音 AI 的成熟使得「AI 替代人工反馈」首次成为可能

**设计响应**：
- 所有核心功能免费（无订阅墙、无注册墙）
- AI 反馈通过三条免费路径实现：本地模型（Ollama）/ 免费网页大模型 API（Gemini / Groq / DeepSeek）/ 本地规则评分兜底，用户按自身情况选择，无需付费
- 即便完全不接入任何 AI，规则评分也能提供有意义的基础反馈，不卡住主流程

---

#### P10 / P12 · 口语练习的孤岛困境

**现象**：大多数学生口语练习路径为——
1. 自言自语（无反馈）
2. 找同学练（互相不知道对错）
3. 报付费外教（高成本，难持续）

产品当前完全没有口语模块，是四科中唯一的空白。

**根因**：口语练习需要「说 → 被听到 → 获得反馈」的闭环，而这条链路在没有外部人的情况下无法形成。直到语音识别（ASR）+ 大语言模型（LLM）的组合成熟后，AI 才能在低成本下扮演「听众 + 评估者」的角色。

**设计响应**（见 7.4 节）：接入语音模型，实现「说话 → 实时转录 → AI 评估 → 结构化反馈」的完整口语练习闭环。

---

#### P3 · 单词孤立，缺乏词族联系

**现象**：背下 `abundant`，看到 `abundance` 仍然卡壳；生词无法从词根推断意思。

**根因**：传统「单词→中文释义」线性结构忽略了派生关系和同义词网络，导致孤立记忆、迁移能力弱。

**现有设计响应**：产品已实现词根-派生词双向图（`derivativeEdges`）和同义词关联，但图形化呈现尚不直观，用户需主动探索才能发现联系。

**改进方向**：词族关系在卡片上直接可见，而非需要点击展开。

---

#### P8 · 写完作文，不知道差在哪里

**现象**：写完 Task 2 的 40 分钟后，无法判断自己的质量，只能等老师，反馈周期 2～7 天，学习动力快速衰减。

**根因**：雅思写作按四维度评分（任务完成、连贯衔接、词汇、语法），缺乏即时、可操作的反馈工具。

**设计响应**：按官方四维标准结构化展示 AI 反馈，定位到原文具体位置，给出可执行的修改建议。

### 7.2 Nielsen 启发式原则映射

| 原则 | 当前状态 | 改进方向 |

## 八、功能需求规格

### 8.1 新手引导（Onboarding）

**解决痛点**：P1（方向不明）、P2（免费可信度疑虑）

**流程**：首次打开产品，触发 3 步引导：
1. **选择备考目标**：目标分数（5.5 / 6.0 / 6.5 / 7.0+）+ 距离考试时间
2. **诊断当前水平**：每科 2 道快速自评题（「我的阅读大约能做对几题？」）
3. **生成第一周计划**：根据目标分数和弱科，推荐今日先做哪个模块

**验收标准**：
- [ ] 引导 3 步内完成，无需注册和登录
- [ ] 引导完成后自动跳转到推荐的第一个任务
- [ ] 可随时从设置页重新进入引导

---

### 8.2 词汇系统（支撑层）

**解决痛点**：P3（词汇孤立）、P4（遗忘时机）、P5（任务滚雪球）

**核心设计**：
- **词族地图**：单词卡片内嵌树状图，直接展示派生词（按词性分组）和同义词，无需点击展开
- **自适应每日目标**：根据词库大小和剩余时间动态调整「今日复习 N 词 + 新学 M 词」，避免积压
- **间隔复习调度**：艾宾浩斯曲线自动计算复习时机，首页显示今日待复习数量

**与四科的联动**：
- 阅读/听力练习时，选中生词 → 浮动工具条 → 一键入库 + 自动打上来源标签（`Reading-P2`）
- 口语练习后，AI 标记的「回避词」自动加入待学列表

---

### 8.3 听力 / 阅读专项

**解决痛点**：P6（生词无法沉淀）、P7（错题缺分析）

**核心设计**：
- **生词即时捕获**：选中文本后 150ms 内出现浮动工具条，「加入词库」「查看释义」「标记生词」
- **错题分析**：每道题答错后展示「考点类型」（推断题/细节题/词汇题）和「错误模式归因」
- **错题词汇联动**：错题中出现的高频生词自动推送到词库待学列表

---

### 8.4 🎙️ 口语教练（核心新增模块）

**解决痛点**：P10（口语孤岛）、P11（发音无反馈）、P12（陪练成本高）

#### 产品形态

AI 口语教练模拟雅思口语考试的真实对话情境，提供三种练习模式：

| 模式 | 说明 |
|------|------|
| **Part 1 自由练** | 从题库随机抽取 Part 1 常见问题，AI 追问，无时限 |
| **Part 2 计时卡片** | 抽取 Cue Card 话题，1 分钟准备 + 2 分钟作答（计时器） |
| **Part 3 深度对话** | AI 扮演考官，围绕 Part 2 话题追问 4～6 个高阶问题 |

#### 技术路径（全链路零成本）

```
用户说话
   │
   ▼
【语音转文本】浏览器 Web Speech API（免费，无需安装任何模型）
   │  实时字幕输出
   ▼
【对话生成 + 质量评估】── 三档可选，全部免费 ──
   ├── 选项 A：本地模型（Ollama + Llama 3 / Mistral）
   │          完全离线，无网络依赖，一次下载永久可用
   ├── 选项 B：免费网页大模型 API
   │          （如 Google Gemini 免费层、Groq 免费层、
   │           DeepSeek 免费额度，均无需付费订阅）
   └── 选项 C：本地规则评分（零依赖兜底方案）
              流利度（停顿次数）· 词汇丰富度（TTR）· 时长达标
   │
   ▼
结构化反馈展示
```

**三档方案对比**：

| 方案 | 安装门槛 | 网络需求 | 反馈质量 | 推荐场景 |
|------|---------|---------|---------|---------|
| 本地模型（Ollama） | 需下载模型（3～8GB） | 无 | 高 | 追求隐私、长期使用 |
| 免费网页 API | 注册账号，填入免费 Key | 需要 | 高 | 快速上手，无需安装 |
| 本地规则评分 | 零安装 | 无 | 基础 | 网络受限或初次体验 |

产品默认引导用户选择「免费网页 API」（门槛最低），并在设置页提供切换入口，不强制绑定任何一种方案。

#### 反馈维度（AI 方案时）

按雅思口语官方四维标准：

| 维度 | 展示内容 |
|------|---------|
| 流利度与连贯性（FC） | 停顿位置标注、连接词使用频率 |
| 词汇丰富度（LR） | 高亮重复词，推荐替换词（自动加入待学词库） |
| 语法多样性（GRA） | 标注语法错误，给出修正版本 |
| 发音（P） | 标注元音/辅音偏差（依赖 ASR 置信度数据） |

## 十一、成本与技术约束

### 11.1 用户侧零成本原则

产品所有功能均可在**无任何付费**的前提下使用，AI 能力通过三条免费路径提供：

| 功能 | 实现方式 | 用户成本 |
|------|---------|---------|
| 间隔复习算法 | 本地计算（无需服务器） | 零 |
| 语音转文本 | 浏览器内置 Web Speech API | 零 |
| 模拟题库 | 本地 JSON / 静态文件 | 零 |
| AI 反馈（兜底） | 本地规则评分（流利度 / TTR / 时长） | 零 |
| AI 反馈（标准） | 免费网页大模型 API（Gemini / Groq / DeepSeek 免费层） | 零（需注册账号） |
| AI 反馈（隐私模式） | 本地模型（Ollama + Llama 3 / Mistral，一次下载） | 零（需下载模型文件 3～8GB） |

**免费网页 API 说明**（截至文档撰写日期，具体额度以各平台当前政策为准）：

| 平台 | 免费额度参考 | 获取难度 |
|------|------------|---------|
| Google Gemini | 每分钟 15 次，每天 1500 次 | 注册 Google 账号即可 |
| Groq | 每日限量，速度极快 | 注册 Groq 账号即可 |
| DeepSeek | 注册赠送 Token | 国内手机号注册 |

> **重要**：免费额度政策由第三方平台决定，随时可能调整。产品设计应确保即便无任何 AI 接入，用户仍能通过规则评分完成有意义的口语练习，而非卡住流程。

### 11.2 技术边界（Out of Scope）

- 云端数据同步（当前为 LocalStorage，多设备不互通）
- 服务器端 AI 代理（避免运营成本）
- 移动端 Native App（当前为响应式 Web）
- 用户账号体系
